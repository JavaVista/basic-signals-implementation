Demo explanation: This demonstrates the basic implementation and core concepts of Signals. A signal represents a reactive value (observable state) that can be read (by calling its function) and updated. In this demo: Clicking the button updates the counter signal, and only UI elements that directly depend on the signal (like the displayed counter) are updated. Also, in the Two-Way Binding example, the input field updates the corresponding signal, reflecting the changes immediately.
